# PRODUCT — sysdesign site

The public landing page for **sysdesign**, a Claude Code plugin that gives an AI coding agent
system-design knowledge. Register is **brand**: design carries the identity. Scope is one page
(`/`) plus a `404`.

## What it sells

A developer tool, in one screen: what the plugin is (1 skill, 11 commands, 14 self-contained
reference files), the method (*state the constraint → pick the option → name the tradeoff*), how
to install and update in Claude Code, and who made it. It links out to GitHub and the maintainer.

## Audience & scene

Developers evaluating a plugin — at a terminal, in a dark editor, skimming for *what is this /
is it real / how do I install*. That scene forces a **dark, terminal-native** treatment, not a
bright SaaS marketing page. The proof element is a fake terminal showing a real
`/sysdesign:compare` answer, because the audience trusts a transcript over a headline.

## Color strategy

**Committed, dark base + a three-node accent triad.** The surface is ink (`#0e1116`); the brand
is a blueprint **blue** (`#5b8def`) primary with **teal** (`#3bb89a`) and **amber** (`#e0a458`)
as secondary accents — the same three nodes as the logo mark. Deliberately *not* the fintech-navy
or AI-neon reflex, and not a warm cream body. Color appears with intent (the triad, the terminal
options), never as decoration.

## Success

The visitor understands the plugin and copies the install command. Everything on the page serves
that: the terminal proof, the command grid, the two-line install, the update instructions.
