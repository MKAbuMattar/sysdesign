# sysdesign site

The marketing landing page for the sysdesign plugin. Agent instructions live in **@AGENTS.md**
— read it first (stack, layout, commands, hard rules).

## Design context (read before any UI work)

- **@PRODUCT.md** — what this site is, its register (`brand`), audience, and color strategy.
- **@DESIGN.md** — the visual system: tokens, type, spacing, motion, components, and the bans.

The plugin itself (the skill + commands) is the parent repo; its editorial system is the
repo-root `DESIGN.md`. This `site/` folder is a separate concern — the *visual* brand — and its
`DESIGN.md` governs here.
